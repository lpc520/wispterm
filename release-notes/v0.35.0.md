# Phantty v0.35.0

> ⚠️ **macOS support is new and may still have bugs — use with care.** This is
> the first release where Phantty runs as a native macOS app (Apple Silicon and
> Intel). The AppKit window lifecycle, Metal renderer, and multi-window paths are
> young and can still misbehave in scenarios we haven't covered yet. If you need
> rock-solid daily stability on macOS, prefer Linux/Windows or
> [Ghostty](https://ghostty.org/) for now. Please report any issues you hit.

## Highlights

- **macOS support (beta).** Phantty now builds and runs as a native `.app` on
  macOS 13+: AppKit windows with a unified title bar, multi-window support, CJK
  font rendering, and color-emoji rendering.
- **macOS: native Cmd / Option shortcuts.** Application shortcuts use Cmd in
  place of Ctrl and Option in place of Alt, matching macOS conventions (e.g.
  `Cmd+Shift+P` opens the command center). `Ctrl+` ` and `Ctrl+Tab` /
  `Ctrl+Shift+Tab` stay on Ctrl to avoid colliding with system shortcuts.
- **macOS: correct split / clip rendering on Metal.** The Metal backend now
  applies per-pane viewport, scissor clipping, blend-mode variants, and sampler
  state per draw. Splits position correctly, content no longer overlaps the
  title bar or sidebar, and color emoji / background images blend correctly.
- **AI Chat tabs restore across sessions.** AI Chat tabs are saved and reopened
  from their persisted agent-history session when you restart Phantty.

## Notes

- Windows behavior is unchanged.
- macOS builds are provided as signed and notarized `.dmg` images for Apple
  Silicon (`aarch64`) and Intel (`x86_64`); see the assets below.
