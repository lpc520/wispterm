# WispTerm v1.14.0

> This release adds Jina-powered web search and web reading for Copilot, a
> cross-environment file copy tool, WeChat inbound media handling, and a Windows
> SSH helper fix for quieter file transfers.

## Added

- **Copilot web search.** Copilot can now use the `websearch` tool, and users can
  dispatch `$websearch` from the composer. Results are powered by Jina, respect
  the configured system proxy transport, and use the `jina-api-key` config value
  when present.
- **Copilot web reader.** Copilot can now use `webread` for URL and local-file
  reading, with `$webread` composer dispatch, background request jobs, transcript
  formatting, and safer UTF-8 truncation.
- **Cached local-file reading.** Local-file `webread` conversions are cached by
  content hash under the conversation working directory, reducing repeated
  upload/conversion work for unchanged files.
- **Cross-environment file copy.** Copilot can copy artifacts between local,
  WSL, and SSH surfaces with `copy_file`, including staging SSH/WSL artifacts
  under `wispterm-files` before sending them as attachments.
- **WeChat inbound files and images.** The direct WeChat bridge can now receive,
  decrypt, download, save, and route inbound file/image media into the active
  Copilot flow.

## Changed

- **OpenSSH config import placement.** The command center entry for importing
  OpenSSH config was moved lower in the list to keep common actions easier to
  reach.
- **Copilot prompt guidance.** The desktop Copilot prompt now advertises
  `copy_file`, `websearch`, and `webread` workflows more explicitly.

## Fixed

- **Hidden SSH askpass helper on Windows.** Password-authenticated SSH/SCP helper
  operations now use a dedicated `wispterm-ssh-askpass.exe` Windows-subsystem
  helper instead of a PowerShell script, preventing console-window flashes when
  Copilot copies files to or from SSH surfaces.
- **Askpass helper packaging.** Portable and installer Windows packages now
  include `wispterm-ssh-askpass.exe` next to `wispterm.exe`, matching the runtime
  lookup path used by SSH/SCP helpers.
- **Consistent SSH helper environment.** SCP transfers, SSH tunnels, remote file
  helpers, and HTML preview SSH commands now share the same askpass environment
  setup.
- **HTML preview teardown.** Preview servers are stopped when the owning preview
  closes.
- **WeChat media privacy and follow-up flow.** Inbound file paths are hidden from
  chat output, and file follow-up instructions now wait for the media save flow
  to finish.

# WispTerm v1.14.0（中文）

> 本版本为 Copilot 新增基于 Jina 的网页搜索和网页读取、跨环境文件复制工具、微信入站
> 文件/图片处理，并修复 Windows SSH helper 传输时的闪屏问题。

## 新增

- **Copilot 网页搜索。** Copilot 现在可以使用 `websearch` 工具，用户也可以从输入框
  发起 `$websearch`。搜索由 Jina 提供，复用系统代理传输，并在配置了
  `jina-api-key` 时自动使用。
- **Copilot 网页读取。** Copilot 现在可以用 `webread` 读取 URL 和本地文件，并支持
  `$webread` 输入框命令、后台请求任务、转录格式化和更安全的 UTF-8 截断。
- **本地文件读取缓存。** 本地文件 `webread` 转换会按内容 hash 缓存在会话工作目录下，
  未变化文件的重复上传/转换会减少。
- **跨环境文件复制。** Copilot 可以用 `copy_file` 在本地、WSL 和 SSH surface 之间复制
  artifact，也可以先把 SSH/WSL 文件暂存到 `wispterm-files` 再作为附件发送。
- **微信入站文件和图片。** 直连微信桥现在可以接收、解密、下载、保存并路由入站
  文件/图片媒体到当前 Copilot 流程。

## 调整

- **OpenSSH config 导入位置。** 命令中心里的 OpenSSH config 导入入口下移，让常用操作
  更容易触达。
- **Copilot 提示词。** 桌面 Copilot prompt 更明确地描述了 `copy_file`、`websearch`
  和 `webread` 的使用流程。

## 修复

- **Windows SSH askpass helper 隐藏运行。** 使用密码认证的 SSH/SCP helper 现在改用
  Windows subsystem 的 `wispterm-ssh-askpass.exe`，不再通过 PowerShell 脚本取密码，
  避免 Copilot 在 SSH surface 之间复制文件时出现控制台闪屏。
- **Askpass helper 打包。** Windows portable 和 installer 包现在会把
  `wispterm-ssh-askpass.exe` 放在 `wispterm.exe` 同目录，和运行时 SSH/SCP helper 查找
  路径保持一致。
- **SSH helper 环境变量统一。** SCP 传输、SSH tunnel、远程文件 helper 和 HTML preview
  的 SSH 命令现在复用同一套 askpass 环境变量设置。
- **HTML 预览清理。** 预览关闭时会停止对应的 preview server。
- **微信媒体隐私和后续流程。** 入站文件路径不会直接暴露在聊天输出里，文件 follow-up
  指令也会等待媒体保存流程完成后再继续。
