# WispTerm v1.15.0

> This release adds the Skill Center for cross-machine skill inventory and sync,
> moves the desktop render loop toward event-driven idle behavior, and improves
> file explorer refresh, remote skill operations, and markdown preview text
> wrapping.

## Added

- **Skill Center.** The command center now includes a Skill Center panel for
  scanning Claude/Codex skills across local, WSL, and SSH sources, comparing
  local-vs-server state, previewing `SKILL.md`, and deploying or importing
  skills with confirmation flows.
- **Async skill operations.** Skill scans, transfers, and remote previews now run
  off the UI thread, with visible busy/error states and SSH stderr summaries when
  an operation fails.
- **Event-driven render gate.** The Windows render loop now draws only when the
  terminal needs a frame, with cross-thread wakeups for PTY output, resize, and
  platform events. Optional frame-latency diagnostics can report input-to-present
  timing and stalled input events.
- **File explorer manual refresh.** File explorer panels now support a clickable
  header refresh button, `Ctrl/Cmd+R`, and `F5`, while preserving selection and
  scroll position where possible.

## Changed

- **Skill Center layout.** The Skill Center evolved from a matrix view into a
  local-library workflow with target pickers, overwrite confirmation, row
  selection, and keyboard shortcuts for preview/deploy actions.
- **macOS local file explorer tracking.** Local file explorer panels on macOS now
  follow the shell's live working directory instead of staying pinned to stale
  paths.
- **Idle rendering behavior.** Command center, new-session, and settings overlays
  are treated as statically-open UI for idle CPU purposes, while their key and
  text handlers explicitly mark the UI dirty when state changes.

## Fixed

- **Overlay key repaint latency.** Command center, new session, and settings
  overlay key handling now repaints immediately in the event-driven loop instead
  of waiting for an incidental wake.
- **Skill sync hangs and crashes.** Local and remote skill operations now drain
  stdout/stderr concurrently, avoid pipe deadlocks, and fix several double-free
  or cleanup hazards in Skill Center confirmation and scan paths.
- **Remote scan feedback.** Unreachable sources now show a clearer "no
  connection" state, and failed sync operations surface a concise stderr summary.
- **File explorer refresh recovery.** Remote rescans clear pending restore state
  when enqueueing fails, avoiding stale selection/scroll restoration.
- **Markdown preview CJK wrapping.** Markdown preview line wrapping now respects
  UTF-8 character boundaries, fixing garbled CJK output at wrap points.
- **macOS test linking.** Native macOS service tests now link the required
  UserNotifications framework.

# WispTerm v1.15.0（中文）

> 本版本新增跨机器技能盘点与同步的 Skill Center，推进桌面端事件驱动空闲渲染，并改进
> 文件浏览器刷新、远端技能操作和 Markdown 预览文本换行。

## 新增

- **Skill Center。** 命令中心现在包含 Skill Center 面板，可以扫描本地、WSL 和 SSH
  来源中的 Claude/Codex skills，比对本地与服务器状态，预览 `SKILL.md`，并通过确认
  流程部署或导入 skills。
- **异步技能操作。** Skill 扫描、传输和远端预览现在会在 UI 线程之外运行，并提供可见的
  忙碌/错误状态；SSH 操作失败时会展示精简后的 stderr 摘要。
- **事件驱动渲染门控。** Windows 渲染循环现在只在终端需要新帧时绘制，并为 PTY 输出、
  resize 和平台事件提供跨线程唤醒。可选帧延迟诊断可以报告 input-to-present 延迟和
  stall 输入事件。
- **文件浏览器手动刷新。** 文件浏览器面板现在支持标题栏刷新按钮、`Ctrl/Cmd+R` 和
  `F5`，并尽量保留选择项和滚动位置。

## 调整

- **Skill Center 布局。** Skill Center 从矩阵视图演进为本地技能库工作流，新增目标选择、
  覆盖确认、行选择，以及预览/部署相关键盘操作。
- **macOS 本地文件浏览器目录跟随。** macOS 本地文件浏览器现在会跟随 shell 的实时工作
  目录，不再停留在过期路径。
- **空闲渲染行为。** 命令中心、新会话和设置页这类静态打开的 overlay 不再让空闲循环
  持续渲染；同时它们的按键和文本处理会在状态变化时显式标记 UI 需要重绘。

## 修复

- **Overlay 按键重绘延迟。** 命令中心、新会话和设置页的按键处理现在会在事件驱动循环
  中立即重绘，不再等到下一次偶然唤醒。
- **Skill 同步卡死和崩溃。** 本地和远端 skill 操作现在会并发 drain stdout/stderr，避免
  管道死锁，并修复 Skill Center 确认和扫描路径中的多处 double-free / 清理风险。
- **远端扫描反馈。** 不可达来源现在会显示更清楚的 "no connection" 状态；同步失败时会
  展示精简后的 stderr 摘要。
- **文件浏览器刷新恢复。** 远端 rescan 入队失败时会清理待恢复状态，避免选择项/滚动位置
  恢复到过期状态。
- **Markdown 预览 CJK 换行。** Markdown 预览现在按 UTF-8 字符边界换行，修复 CJK 文本
  在换行点乱码的问题。
- **macOS 测试链接。** 原生 macOS services 测试现在会链接所需的 UserNotifications
  framework。
