# WispTerm v1.17.0

> This release brings up the experimental Linux desktop build, adds an SSH port
> forwarding manager, expands Copilot's terminal/REPL abilities, and adds a
> PubMed literature search tool.

## Added

- **Experimental Linux desktop build.** WispTerm now has an SDL3/OpenGL Linux
  host with fontconfig discovery, CJK fallback, clipboard/cursor/file-dialog
  services, IME preedit support, and custom titlebar hit testing.
- **Linux AppImage release asset.** Tag releases now build and upload a portable
  `WispTerm-*-x86_64.AppImage` for community testing.
- **SSH Port Forwarding manager.** The command center includes a Port Forwarding
  tab for saved SSH profiles, with local and reverse forwarding rules, helper
  process lifecycle management, persisted rule storage, and a dedicated form UI.
- **PubMed Copilot tool.** Copilot can search PubMed, fetch abstracts, and format
  biomedical literature results through the new `pubmed` tool and `$pubmed`
  background command.
- **Claude Code/Codex prompt answering.** Copilot can inspect live terminal
  prompts and answer approval/menu choices through `terminal_answer_prompt`
  instead of blindly sending keys.

## Changed

- **Human-like REPL execution.** Python/R/Node-style line REPL input is typed as
  a user would type it, without wrapper/sentinel code, and waits for the prompt
  to return before treating output as complete.
- **Live snapshot handling.** Copilot terminal snapshots now keep the visible
  screen tail while bounding history size; web-remote and Jupyter paths keep the
  fuller snapshot behavior they need.
- **Distill suggestions are opt-in.** Automatic skill-distill suggestions are
  now gated by `ai-distill-suggest`, which defaults to `false`.

## Fixed

- **Remote/Jupyter snapshot regressions.** Web-remote and Jupyter URL paths keep
  their full snapshot history instead of inheriting the smaller Copilot tail cap.
- **Port forwarding robustness.** Rule loading, helper identity, manager
  snapshots, app startup, helper lifecycle races, and form rendering bounds were
  hardened.
- **Command center child escape handling.** Escape behavior in child command
  center surfaces is corrected.
- **PubMed parsing accuracy.** Abstract extraction is scoped to the primary
  `<Abstract>` element, and publication years come from `PubDate`.

## Known limitations

- **No macOS x86_64 DMG for this release.** v1.17.0 intentionally skips the
  manual Intel macOS workflow and publishes the Apple Silicon macOS DMG only.
- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.

# WispTerm v1.17.0（中文）

> 本版本带来实验性的 Linux 桌面构建，新增 SSH 端口转发管理器，扩展 Copilot 的
> terminal/REPL 能力，并加入 PubMed 文献检索工具。

## 新增

- **实验性 Linux 桌面构建。** WispTerm 现在具备 SDL3/OpenGL Linux host，支持
  fontconfig 字体发现、CJK fallback、剪贴板/光标/文件对话框服务、IME 预编辑，以及自定义
  标题栏命中测试。
- **Linux AppImage 发布资产。** tag release 现在会构建并上传便携的
  `WispTerm-*-x86_64.AppImage`，用于社区测试。
- **SSH 端口转发管理器。** 命令中心新增 Port Forwarding 标签页，可基于已保存 SSH
  profile 管理本地/反向转发规则、helper 进程生命周期、持久化规则和专用表单 UI。
- **PubMed Copilot 工具。** Copilot 现在可以通过新的 `pubmed` 工具和 `$pubmed`
  后台命令搜索 PubMed、抓取摘要并格式化生物医学文献结果。
- **Claude Code/Codex prompt 回答。** Copilot 可以检查实时终端 prompt，并通过
  `terminal_answer_prompt` 回答审批/菜单选项，而不是盲目发送按键。

## 调整

- **更接近人工输入的 REPL 执行。** Python/R/Node 这类行式 REPL 现在按用户实际输入的
  方式发送代码，不再包裹 wrapper/sentinel，并等待 prompt 返回后再判定输出完成。
- **实时快照处理。** Copilot 终端快照会保留可见屏幕尾部并限制历史大小；web-remote 与
  Jupyter 路径保留所需的更完整快照行为。
- **Distill 建议改为显式开启。** 自动 skill-distill 建议现在由 `ai-distill-suggest`
  控制，默认值为 `false`。

## 修复

- **Remote/Jupyter 快照回归。** web-remote 和 Jupyter URL 路径会保留完整快照历史，不再
  继承较小的 Copilot tail cap。
- **端口转发稳定性。** 强化了规则加载、helper 身份、manager 快照、应用启动、helper
  生命周期竞态和表单渲染边界。
- **命令中心子界面 Escape 行为。** 修正 command center 子界面的 Escape 处理。
- **PubMed 解析准确性。** 摘要提取现在限定在主 `<Abstract>` 元素，年份来自 `PubDate`。

## 已知限制

- **本次不发布 macOS x86_64 DMG。** v1.17.0 会跳过手动 Intel macOS workflow，只发布
  Apple Silicon macOS DMG。
- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
