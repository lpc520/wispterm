# WispTerm v1.19.0

> This release makes preview panes part of the split-tree workspace, improves
> terminal/render responsiveness, and adds a Windows portable-compat package
> with a bundled modern ConPTY for older Windows 10 machines.

## Added

- **Preview panes as first-class split panes.** Preview panes now live in the
  split tree beside terminals, can be created or reused from preview commands,
  participate in focus, wheel scroll/zoom, Alt-drag swapping, and session
  save/restore.
- **Windows portable-compat package.** The Windows release now publishes
  `wispterm-windows-portable-compat-v1.19.0.zip`, replacing the old WebView2-only
  flavor. It bundles `WebView2Loader.dll` plus `conpty.dll` and
  `OpenConsole.exe` so Codex/Claude Code-style TUI apps get modern mouse and
  scrollbar behavior on old inbox ConPTY hosts.
- **`windows-conpty` config key.** Windows users can choose the console host
  policy with `windows-conpty = auto` or `windows-conpty = system`.

## Changed

- **Preview UI is tiling-only.** The old right-dock preview model and dead
  header close button were removed in favor of split panes.
- **Rendering is leaner.** UI glyphs/quads are batched into one instanced
  pipeline; active-tab and split-pane dirty gates reduce redundant work; cell
  uploads, underline spans, and sync-output handling avoid unnecessary churn.
- **PTY output is coalesced.** Terminal output reads are batched to reduce wakeups
  without delaying visible updates.
- **Update selection follows the new Windows flavors.** Auto-update package
  matching now prefers the compat flavor when appropriate.

## Fixed

- **Windows DXGI black-screen fallback.** The flip-present path now detects
  silently broken drivers and falls back instead of leaving a black window.
- **macOS worker windows.** Worker-thread windows no longer pump `NSApp` events
  from the wrong thread.
- **Glyph atlas repainting.** The render gate repaints when an atlas contains
  glyphs the GPU has not uploaded yet.
- **Viewport wheel scrolling.** Mouse-wheel viewport scrolling marks the UI dirty
  so the repaint follows the input.
- **WSL terminal environment.** Windows sessions bridge the terminal environment
  into WSL commands.
- **Preview close behavior.** `Ctrl+Shift+W` closes the preview pane instead of
  accidentally closing the focused terminal.
- **AI history resume diagnostics.** Resume failures now report clearer causes.

## Known limitations

- **No macOS x86_64 DMG for this release.** v1.19.0 intentionally skips the
  manual Intel macOS workflow and publishes the Apple Silicon macOS DMG only.
- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.

# WispTerm v1.19.0（中文）

> 本版本将 preview pane 纳入 split-tree 工作区，改进终端/渲染响应性，并新增带有现代
> ConPTY 的 Windows portable-compat 包，面向旧版 Windows 10 机器。

## 新增

- **Preview pane 成为一等 split pane。** Preview pane 现在和终端一样位于 split tree
  中，可以由预览命令创建或复用，并参与焦点、滚轮滚动/缩放、Alt 拖拽交换和 session
  保存/恢复。
- **Windows portable-compat 包。** Windows release 现在发布
  `wispterm-windows-portable-compat-v1.19.0.zip`，替代旧的 WebView2-only 变体。
  它会打包 `WebView2Loader.dll`、`conpty.dll` 和 `OpenConsole.exe`，让旧版 inbox
  ConPTY 上的 Codex/Claude Code 这类 TUI 获得现代鼠标和滚动条行为。
- **`windows-conpty` 配置项。** Windows 用户可以通过 `windows-conpty = auto` 或
  `windows-conpty = system` 选择 console host 策略。

## 调整

- **Preview UI 只保留平铺模式。** 旧的右侧 dock preview 模型和失效的 header close
  button 已移除，统一改为 split pane。
- **渲染路径更轻。** UI glyph/quad 现在合并到一个 instanced pipeline；active-tab 和
  split-pane dirty gate 会减少重复工作；cell upload、underline span 和 sync-output
  处理也避免了不必要的开销。
- **PTY 输出读取合并。** 终端输出读取会合并批次，减少 wakeup，同时不延迟可见更新。
- **更新包选择跟随新的 Windows 变体。** 自动更新包匹配现在会在合适时优先选择 compat
  变体。

## 修复

- **Windows DXGI 黑屏回退。** flip-present 路径现在会检测静默失效的驱动，并回退而不是留下黑屏窗口。
- **macOS worker window。** worker 线程窗口不再从错误线程 pump `NSApp` 事件。
- **Glyph atlas 重绘。** 当 atlas 中有尚未上传到 GPU 的 glyph 时，render gate 会触发重绘。
- **Viewport 滚轮滚动。** 鼠标滚轮 viewport scrolling 现在会标记 UI dirty，让重绘紧跟输入。
- **WSL 终端环境。** Windows session 会把终端环境传给 WSL 命令。
- **Preview 关闭行为。** `Ctrl+Shift+W` 现在关闭 preview pane，而不是误关当前终端。
- **AI history resume 诊断。** 恢复失败时现在会给出更清晰的原因。

## 已知限制

- **本次不发布 macOS x86_64 DMG。** v1.19.0 会跳过手动 Intel macOS workflow，只发布
  Apple Silicon macOS DMG。
- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
