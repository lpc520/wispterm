# WispTerm v1.19.1

> Patch release: fixes the v1.19.0 black-screen regression on Windows machines
> with weak integrated GPUs (typically Win11 thin-and-light laptops).

## Fixed

- **Windows black screen on weak-iGPU laptops (v1.19.0 regression).** The
  v1.19.0 present-path guard could switch a window from the DXGI flip-model
  presenter to GDI in the middle of a session. Windows does not support that
  transition — a window that has presented through a flip-model swapchain
  cannot revert to GDI — so machines that were merely slow on v1.18.0 went
  black on v1.19.0. The slow-present watchdog now keeps the session on the
  flip path and instead records a marker so the *next* launch presents via
  GDI from the very first frame. On affected laptops the first launch after
  upgrading may still feel slow once; from the second launch onward both the
  black screen and the v1.18-era lag are gone.
- **False alarms from the present verification probe.** The first-frames
  content probe required bit-exact pixel matches, which misfired on 10-bit /
  dithering display pipelines and pushed perfectly working machines onto the
  broken fallback. It now tolerates small rounding differences while still
  catching real failures.
- **Guaranteed recovery after a hard present failure.** When the flip path
  fails outright mid-session, the app still attempts the GDI fallback as a
  best effort, and now also records the marker so the next launch is
  guaranteed to start on the working GDI path.

## Notes

- Setting `wispterm-d3d-present = false` remains available as a manual
  override for the flip-model presenter.
- Upgrading to a newer WispTerm version retries the flip path once, so fixed
  drivers can return to the faster presenter automatically.
