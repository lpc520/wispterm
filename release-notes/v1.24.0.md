# WispTerm v1.24.0

> This release adds **in-session AI model switching** with context handoff,
> makes the **Copilot sidebar easier to discover**, and fixes tmux/SSH preview
> path handling, tmux control-mode reply routing, and two Intel macOS crash
> paths. It also refreshes the docs, FAQ, and release packaging notes.

## Added

- **Switch AI models without starting over.** AI Chat tabs and the Copilot
  sidebar now support `/model` to open a saved-profile picker, `/model <name>`
  to switch directly, and `/模型` as a Chinese alias. Clicking the model label
  in the panel header opens the same switcher. After a switch, WispTerm asks the
  new model to summarize the prior transcript in the background and collapses
  the old history into a **Conversation summary** card; if the summary fails,
  the full raw history is kept. The switch is session-only and does not change
  the global default profile. (#251)
- **Copilot edge handle.** The terminal's right edge now reveals a subtle
  Copilot handle when the pointer nears it; hovering shows the live shortcut,
  and clicking toggles the Copilot sidebar. New users see a one-time shimmer,
  gated by the persisted `copilot-hint-shown` flag. The handle can be disabled
  with `copilot-hint = false`. (#249)
- **Native Copilot entry points.** Windows/Linux titlebars now include a Copilot
  icon, macOS gets a native **View -> Toggle Copilot** menu item, and the
  command center plus shortcuts overlay list **Toggle Copilot**. (#249)

## Changed

- **macOS Intel release packaging.** The Intel (`x86_64`) DMG workflow now runs
  automatically after the tag-triggered Apple Silicon release workflow succeeds,
  so normal releases no longer require a separate manual Intel packaging step.
  (#250)
- **Docs and FAQ refresh.** The website, embedded AI docs, README, GitHub wiki
  FAQ, and release documentation now describe `/model`, the Copilot discovery
  affordances, experimental Linux AppImage availability, and current release
  assets.

## Fixed

- **tmux over SSH preview paths.** Ctrl/Cmd-click preview, download, and open
  actions for relative paths inside plain `tmux` over SSH now query
  `tmux display-message -p '#{pane_current_path}'` when WispTerm can detect a
  tmux-like alternate-screen session. If the query fails, the old OSC 7 cwd
  fallback remains unchanged. (#247)
- **tmux control-mode reply routing.** Guarded command replies from tmux
  control mode are no longer mistaken for capture-pane output, so split-window
  replies are not written into panes as stray `%` text. (#248)
- **macOS Intel FreeType crash.** Intel macOS builds now pass
  `FT_LOAD_NO_AUTOHINT` through every glyph load path, avoiding the FreeType
  CJK autofitter crash seen when rendering CJK text, titlebar text, or the
  session launcher on some machines. (#250)
- **macOS Metal teardown crash.** Quitting the last macOS window no longer
  tears down Metal pipeline objects during process exit, avoiding a crash in
  manually-retained bridge objects while still releasing pipelines for
  secondary windows. (#250)
- **Copilot handle repaint and hit-test polish.** Edge-handle animation frames,
  scrollbar coexistence, and caption drag-region handling were tightened so the
  hint does not steal unrelated UI interactions. (#249)

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **macOS remote access remains early.** The macOS relay transport is available
  but still needs more end-to-end field testing.

# WispTerm v1.24.0（中文）

> 本版本新增 **AI 会话内切换模型** 与上下文交接，让 **Copilot 侧栏更容易被发现**，
> 并修复 tmux/SSH 预览路径、tmux 控制模式回复路由，以及两个 Intel macOS 崩溃路径。
> 同时刷新了 docs、FAQ 与发布打包说明。

## 新增

- **不中断会话切换 AI 模型。** AI Chat 标签和 Copilot 侧栏现在支持用 `/model` 打开
  已保存 profile 选择器，用 `/model <name>` 直接切换，并支持中文别名 `/模型`。点击
  面板头部的模型标签也会打开同一个切换入口。切换后，WispTerm 会让新模型在后台总结
  此前对话，并把旧历史折叠成 **Conversation summary / 上文摘要** 卡片；如果摘要失败，
  则保留完整原始历史。该切换只作用于当前会话，不会修改全局默认 profile。（#251）
- **Copilot 边缘握柄。** 当鼠标靠近终端右边缘时，现在会显示一个轻量 Copilot 握柄；
  悬停显示实时快捷键，点击即可打开或关闭 Copilot 侧栏。新用户会看到一次性微闪提示，
  由持久化的 `copilot-hint-shown` 标记控制。可通过 `copilot-hint = false` 关闭该入口。
  （#249）
- **原生 Copilot 入口。** Windows/Linux 标题栏加入 Copilot 图标，macOS 加入原生
  **View -> Toggle Copilot** 菜单项，命令中心与快捷键浮层也会列出 **Toggle Copilot**。
  （#249）

## 调整

- **macOS Intel 发布打包。** Intel (`x86_64`) DMG workflow 现在会在 tag 触发的
  Apple Silicon 发布 workflow 成功后自动运行，常规发布不再需要单独手动触发 Intel 打包。
  （#250）
- **Docs 和 FAQ 刷新。** 网站、内嵌 AI 文档、README、GitHub wiki FAQ 与发布说明现在
  都补充了 `/model`、Copilot 可发现入口、实验性 Linux AppImage 以及当前 release assets。

## 修复

- **tmux over SSH 预览路径。** 在 SSH 里运行普通 `tmux` 时，对相对路径进行
  Ctrl/Cmd-click 预览、下载或打开，现在会在检测到类似 tmux 的 alternate-screen 会话时查询
  `tmux display-message -p '#{pane_current_path}'`。如果查询失败，则继续使用旧的 OSC 7 cwd
  回退逻辑。（#247）
- **tmux 控制模式回复路由。** tmux control mode 的 guarded command reply 不再被误认为
  capture-pane 输出，因此 split-window 回复不会以杂散 `%` 文本写进窗格。（#248）
- **macOS Intel FreeType 崩溃。** Intel macOS 构建现在在所有 glyph load 路径上都传入
  `FT_LOAD_NO_AUTOHINT`，避开某些机器上渲染 CJK 文本、标题栏或 session launcher 时触发的
  FreeType CJK autofitter 崩溃。（#250）
- **macOS Metal 退出崩溃。** 关闭最后一个 macOS 窗口时，不再在进程退出阶段销毁 Metal
  pipeline 对象，避免手动 retain/release 的 bridge 对象崩溃；关闭非最后窗口时仍会正常释放。
  （#250）
- **Copilot 握柄绘制与命中测试。** 收紧了边缘握柄的动画帧、与滚动条共存、以及标题栏
  caption drag 区域处理，避免提示入口抢走无关 UI 交互。（#249）

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
- **macOS 远程访问仍为早期功能。** macOS 中继传输已经可用，但仍需要更多端到端实际验证。
