# WispTerm v1.26.0

> This release fixes Copilot sidebar transcript copying, adds deeper preview
> diagnostics for SSH/browser/Markdown preview paths, and makes the Windows
> graphics bring-up fallback reliable on a fresh profile.

## Added

- **Preview diagnostics logging.** SSH browser tunnels, local HTML serving,
  Markdown preview rendering, and preview-pane open paths now emit more useful
  diagnostics, making remote preview failures easier to investigate from support
  logs.

## Fixed

- **Copilot sidebar transcript copy works from the focused sidebar.** Copy
  shortcuts now route through the Copilot sidebar selection path instead of
  falling back to terminal selection handling.
- **Windows first-launch graphics recovery is reliable.** The DXGI bring-up
  crash fuse now creates the WispTerm config directory before persisting its
  marker, so a fresh profile can still recover by falling back to GDI on the
  next launch after a driver bring-up crash.

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **macOS remote access remains early.** The macOS relay transport is available
  but still needs more end-to-end field testing.

# WispTerm v1.26.0（中文）

> 本版本修复 Copilot 侧边栏记录复制，强化 SSH/browser/Markdown 预览路径的诊断日志，
> 并让 Windows 图形初始化失败后的回退机制在全新配置中也能可靠生效。

## 新增

- **预览诊断日志。** SSH browser tunnel、本地 HTML 服务、Markdown 预览渲染以及预览面板
  打开路径现在会输出更有用的诊断信息，便于从支持日志中排查远程预览失败。

## 修复

- **Copilot 侧边栏获得焦点时可以正确复制记录。** 复制快捷键现在会走 Copilot 侧边栏的
  选择路径，而不是错误回落到终端选择处理。
- **Windows 首次启动图形回退更可靠。** DXGI bring-up crash fuse 在持久化标记前会先创建
  WispTerm 配置目录，因此全新 profile 遇到驱动初始化崩溃后，下次启动也能回退到 GDI。

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
- **macOS 远程访问仍为早期功能。** macOS 中继传输已经可用，但仍需要更多端到端实际验证。
