# WispTerm v1.30.0

> This release adds `wisptermctl spawn` for opening new tabs in the running
> instance, continues the UI-state boundary cleanup, and fixes terminal input,
> history, Copilot setup, Weixin, and terminal I/O reliability issues.

## Added

- **`wisptermctl spawn` (#307).** The control CLI can now open a new tab in the
  running WispTerm instance. It supports `--cwd DIR`, an optional command after
  `--`, and defaults to the active tab's cwd plus the configured shell.
- **Structural debt guards.** Source-scan ratchets now freeze file size,
  monolith globals, AppWindow import-hub re-exports, and remaining direct dirty
  writes so those counts can keep shrinking.

## Changed

- **UI-state boundaries keep moving behind explicit effects.** More input,
  overlay, and AppWindow paths now route state changes through feature-owned
  state and `UiEffect` sinks instead of scattered globals.
- **Terminal I/O internals were hardened.** Mailbox payload writes, resize/control
  messages, surface I/O lifecycle state, subprocess capture, and persisted state
  writes now have clearer ownership and safer failure behavior.

## Fixed

- **Shift+Enter in Kitty-protocol TUIs (#302).** Shift+Enter now inserts a
  newline instead of being encoded as the wrong key sequence.
- **Queued terminal writes.** Control messages no longer evict pending writes
  from the terminal mailbox.
- **Copilot setup.** The Copilot API setup prompt now opens and closes through
  the intended UI path.
- **AI history count clipping.** History counts no longer clip in the renderer.
- **Weixin/session center flow.** Weixin acknowledgements include the `/help`
  hint, and Space previews session transcripts from the session center.
- **Skill install safety.** Installing skills from GitHub now rejects archive
  entries that try to escape the target directory.

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **macOS remote access remains early.** The macOS relay transport is available
  but still needs more end-to-end field testing.

# WispTerm v1.30.0（中文）

> 本版本新增 `wisptermctl spawn`，可在运行中的 WispTerm 实例里打开新 tab；
> 同时继续推进 UI state 边界清理，并修复终端输入、history、Copilot 设置、
> Weixin 和终端 I/O 可靠性问题。

## 新增

- **`wisptermctl spawn`（#307）。** 控制 CLI 现在可以在运行中的 WispTerm 实例里打开新
  tab。它支持 `--cwd DIR`，支持在 `--` 后指定要运行的命令；未指定时使用 active tab 的 cwd
  和配置里的默认 shell。
- **结构债 guard。** 新的 source-scan ratchet 会冻结文件大小、monolith 全局状态、
  AppWindow import-hub re-export，以及剩余直接 dirty 写入数量，后续只能继续下降。

## 调整

- **UI state 边界继续收敛到显式 effect。** 更多 input、overlay 和 AppWindow 路径现在通过
  feature-owned state 和 `UiEffect` sink 落地状态变化，而不是散落写全局变量。
- **终端 I/O 内部更稳。** Mailbox payload 写入、resize/control 消息、Surface I/O 生命周期、
  子进程输出捕获和持久化状态写入都进一步明确了所有权和失败处理。

## 修复

- **Kitty protocol TUI 里的 Shift+Enter（#302）。** Shift+Enter 现在会插入换行，不再编码成
  错误按键序列。
- **排队中的终端写入。** control message 不再从 terminal mailbox 中挤掉待写入的数据。
- **Copilot 设置。** Copilot API 设置 prompt 现在通过预期的 UI 路径打开和关闭。
- **AI history 数量裁切。** History 数字在渲染器里不再被裁切。
- **Weixin/session center 流程。** Weixin ack 会带上 `/help` 提示；session center 里按 Space
  可以预览 transcript。
- **Skill 安装安全。** 从 GitHub 安装 skill 时会拒绝试图逃出目标目录的 archive entry。

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
- **macOS 远程访问仍为早期功能。** macOS 中继传输已经可用，但仍需要更多端到端实际验证。
