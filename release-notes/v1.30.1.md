# WispTerm v1.30.1

> Patch release for two fixes after v1.30.0: AI Chat title generation across
> all profiles, and Windows agent-control socket transport reliability.

## Fixed

- **AI Chat auto-title profiles (#364).** Auto-title generation now runs for
  every compatible profile instead of only profiles named `DeepSeek`.
- **Windows agent-control transport (#365).** `wisptermctl` now handles Windows
  overlapped socket reads/writes correctly, fixing command transport failures.

## Tests

- **Windows socket round trip (#366).** CI now exercises the agent-control
  socket path on Windows so this transport regression stays covered.

## Known limitations

- **Linux is experimental.** The Linux AppImage is published for community
  testing and is not yet considered stable.
- **macOS remote access remains early.** The macOS relay transport is available
  but still needs more end-to-end field testing.

# WispTerm v1.30.1（中文）

> 这是 v1.30.0 之后的补丁版本，修复 AI Chat 在所有 profile 下生成标题的问题，
> 并修复 Windows 上 agent-control socket 传输可靠性。

## 修复

- **AI Chat 自动标题 profile（#364）。** 自动标题现在会作用于所有兼容 profile，
  不再只匹配名为 `DeepSeek` 的 profile。
- **Windows agent-control 传输（#365）。** `wisptermctl` 现在正确处理 Windows
  overlapped socket 读写，修复命令传输失败。

## 测试

- **Windows socket round trip（#366）。** CI 现在会在 Windows 上覆盖 agent-control
  socket 路径，防止这个传输回归再次漏掉。

## 已知限制

- **Linux 仍为实验性质。** Linux AppImage 用于社区测试，尚不视为稳定版本。
- **macOS 远程访问仍为早期功能。** macOS 中继传输已经可用，但仍需要更多端到端实际验证。
