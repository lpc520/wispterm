# WispTerm v1.5.1

> Hotfix for AI agent terminal tools controlling Codex and Claude Code inside
> WSL/SSH panes.

## Fixed

- **Codex / Claude Code shell-wrapper leakage.** `wsl_session_exec` and
  `ssh_session_exec` now refuse to run inside active Codex / Claude Code panes,
  preventing WispTerm sentinel shell wrappers from being pasted into full-screen
  agent apps.
- **Starting full-screen agent apps.** Launching Codex or Claude Code from an
  already-open terminal is now guided through `terminal_repl_exec repl=plain`
  instead of shell exec tools, so startup does not leave command sentinels on the
  screen.
- **Codex prompt submission.** `terminal_repl_exec repl=codex` submits idle
  Codex input with LF, matching Codex's accepted Enter behavior, while still
  using Tab to queue input when Codex is already running.
- **Codex / Claude Code result timing.** REPL sends now wait for the app to
  settle, request approval/input, complete, fail, or hit the configured timeout
  before reporting a result.

# WispTerm v1.5.1（中文）

> 修复 AI agent terminal tools 在 WSL/SSH 面板里控制 Codex 与 Claude Code 时的
> hotfix。

## 修复

- **Codex / Claude Code shell wrapper 泄漏。** `wsl_session_exec` 和
  `ssh_session_exec` 现在会拒绝在活跃的 Codex / Claude Code 面板里运行，避免
  WispTerm 的 sentinel shell wrapper 被粘进全屏 agent app。
- **启动全屏 agent app。** 在已打开的终端里启动 Codex 或 Claude Code 时，现在会引导
  使用 `terminal_repl_exec repl=plain`，不再走 shell exec 工具，避免启动界面残留命令
  sentinel。
- **Codex prompt 提交。** `terminal_repl_exec repl=codex` 在 Codex 空闲时用 LF 提交
  输入，匹配 Codex 接受的 Enter 行为；Codex 正在运行时仍用 Tab 排队输入。
- **Codex / Claude Code 结果等待。** REPL 发送现在会等待 app 稳定、请求确认/输入、
  完成、失败，或达到配置的超时时间后再返回结果。
