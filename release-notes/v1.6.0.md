# WispTerm v1.6.0

> Session restore comes to the Settings page: turn on **Restore tabs on
> startup** and WispTerm brings back your tab and split-pane layout the next
> time you launch. Plus the window now remembers its size, the first-launch AI
> setup form no longer nags, and OSC 52 clipboard writes (e.g. Claude Code's
> copy) are honored.

## Added

- **Restore tabs/layout on startup.** The session-restore engine is now exposed
  as a Settings toggle (**Restore tabs on startup**, after *Language*) instead
  of being config-file-only. With it on, WispTerm reopens your previous tabs,
  split layout (orientation, ratios, focused/zoomed pane), local-shell working
  directories, and AI Chat tabs when you relaunch. Also documented in the
  default config template as `restore-tabs-on-startup`.
- **Remember window size.** The window size is now persisted across launches.
  An explicit size in your config still wins, and cascaded/secondary windows
  keep the default placement.
- **First-launch-only AI setup.** The AI-agent setup form now appears only on
  the very first launch (when no AI profile exists), instead of prompting on
  every start.

## Fixed

- **OSC 52 clipboard writes are honored.** Programs that copy via OSC 52 — such
  as Claude Code's "sent N chars via OSC 52" — now actually populate the system
  clipboard instead of being silently dropped.
- **Window position when closed while maximized.** Closing a maximized window
  now persists its real windowed position instead of `(0,0)`, so it reopens
  where you left it.

## Known limitations

- **WSL / SSH sessions are not restored yet.** Restore currently brings back
  the tab/split layout and local shells only. WSL panes come back as a plain
  local shell (not re-entering the WSL distro), and SSH panes are not faithfully
  restored — passwords are never saved, so any reconnection would require manual
  re-authentication. Faithful WSL/SSH restore is planned for a future release.

# WispTerm v1.6.0（中文）

> 会话恢复进入设置页：打开 **启动时恢复标签页**，下次启动 WispTerm 时会自动恢复你的
> 标签页与分屏布局。此外，窗口现在会记住大小，首次启动的 AI 设置表单不再反复弹出，
> OSC 52 剪贴板写入（例如 Claude Code 的复制）也能正确生效。

## 新增

- **启动时恢复标签页/布局。** 会话恢复引擎现在以设置开关（**启动时恢复标签页**，位于
  *语言* 之后）的形式暴露，而不再只能改配置文件。开启后，重新启动时 WispTerm 会重开
  之前的标签页、分屏布局（方向、比例、聚焦/放大的面板）、本地 shell 的工作目录，以及
  AI Chat 标签页。默认配置模板中也补充了 `restore-tabs-on-startup` 说明。
- **记住窗口大小。** 窗口大小现在会跨启动持久化。配置中显式设置的大小仍优先生效，
  级联/次级窗口保持默认位置。
- **AI 设置仅首次启动。** AI agent 设置表单现在只在首次启动（且尚无 AI 配置）时出现，
  不再每次启动都弹出。

## 修复

- **正确处理 OSC 52 剪贴板写入。** 通过 OSC 52 复制的程序——例如 Claude Code 的
  “sent N chars via OSC 52”——现在会真正写入系统剪贴板，而不再被静默丢弃。
- **最大化状态下关闭时的窗口位置。** 在最大化状态下关闭窗口，现在会持久化其真实的
  窗口位置，而不是 `(0,0)`，因此下次会在你离开时的位置重开。

## 已知限制

- **暂不恢复 WSL / SSH 会话。** 目前恢复仅会还原标签页/分屏布局和本地 shell。WSL 面板
  会以普通本地 shell 的形式恢复（不会重新进入 WSL 发行版），SSH 面板也不会被完整恢复
  ——密码从不保存，任何重连都需要手动重新认证。完整的 WSL/SSH 恢复计划在后续版本中
  支持。
