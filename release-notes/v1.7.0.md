# WispTerm v1.7.0

> AI History is now a full browser for Claude Code and Codex sessions: WispTerm
> scans local and SSH-backed history in the background, streams results while it
> works, groups sessions by provider and date, and reopens cached results
> immediately. This release also tightens session restore, quake-window defaults,
> and release packaging.

## Added

- **AI History browser.** WispTerm can now discover Claude Code and Codex
  history, render it as an AI History tab, and resume sessions from the browser.
- **Background history scanning.** Local and remote scans, transcript previews,
  and result publishing now run through worker jobs so the UI stays responsive
  while history is loading.
- **Incremental and cached results.** History rows stream in batches during a
  scan, and cached metadata is shown immediately on reopen before the
  authoritative scan reconciles the list.
- **Provider and date navigation.** AI History now has provider/category
  filtering, date buckets, per-day navigation, keyboard cycling, and mouse/wheel
  interactions for narrowing large history lists.
- **Remote history cache support.** SSH-backed scans can reuse cached remote
  metadata and update that cache after a fresh scan.
- **Settings polish.** Quake mode now defaults off, quake window geometry is
  persisted, and settings include a restore-defaults action.
- **macOS Intel release workflow.** The automatic tag release still publishes
  Apple Silicon macOS builds, and an on-demand workflow can now attach the Intel
  DMG to the same release.

## Fixed

- **Saved SSH and AI tabs restore correctly.** Session restore now preserves the
  metadata needed to bring back SSH and AI tabs instead of degrading them to
  plain local shells.
- **AI History streaming actually reaches the UI.** The scan job now forwards
  its scan sink correctly, so the browser shows live scan progress and rows.
- **Date filters apply to the rendered list.** Selecting a date bucket now
  filters the visible session list instead of only updating internal state.
- **History cache ownership.** Cache parsing no longer aliases a freed file
  buffer, fixing an intermittent invalid-memory bug after loading cached rows.
- **Thread shutdown and stale results.** AI History workers now guard shared
  state with the session mutex, supersede stale transcript loads, and join
  cleanly during teardown.
- **macOS render diagnostics.** Render diagnostics now guard null OpenGL function
  pointers when running on the macOS Metal backend.

## Known limitations

- **AI History depends on available history files.** Empty, moved, or
  permission-restricted Claude Code/Codex history directories will show fewer or
  no sessions until those sources become available.
- **macOS Intel DMG is manual.** The Intel DMG is built by the on-demand
  workflow after the tag-triggered release creates the GitHub release.

# WispTerm v1.7.0（中文）

> AI History 现在成为完整的 Claude Code / Codex 会话浏览器：WispTerm 会在后台扫描
> 本地和 SSH 远端历史，扫描时持续流式显示结果，按 provider 与日期分组，并在再次打开时
> 立即显示缓存结果。本版本也加强了会话恢复、quake 窗口默认值与发布打包流程。

## 新增

- **AI History 浏览器。** WispTerm 现在可以发现 Claude Code 和 Codex 历史记录，将其渲染
  为 AI History 标签页，并从浏览器中恢复会话。
- **后台历史扫描。** 本地/远端扫描、转录预览和结果发布现在通过 worker job 执行，加载
  历史时 UI 仍保持响应。
- **增量结果与缓存结果。** 扫描过程中历史行会分批流式出现；再次打开时会先显示缓存的
  元数据，再由权威扫描结果进行校准。
- **Provider 与日期导航。** AI History 增加了 provider/category 过滤、日期分桶、按天
  导航、键盘切换以及鼠标/滚轮交互，便于筛选大量历史会话。
- **远端历史缓存。** SSH 远端扫描可以复用缓存的远端元数据，并在完成新扫描后更新缓存。
- **设置体验完善。** Quake 模式现在默认关闭，quake 窗口位置与大小会持久化，设置页也
  增加了恢复默认值操作。
- **macOS Intel 发布工作流。** 自动 tag 发布仍会发布 Apple Silicon macOS 构建；新增的
  按需工作流可以把 Intel DMG 附加到同一个 release。

## 修复

- **正确恢复已保存的 SSH 和 AI 标签页。** 会话恢复现在会保留恢复 SSH / AI 标签所需的
  元数据，不再退化为普通本地 shell。
- **AI History 流式结果会真正到达 UI。** 扫描 job 现在会正确转发 scan sink，因此浏览器
  能显示实时扫描数量和历史行。
- **日期过滤作用到可见列表。** 选择日期分桶后，现在会过滤实际渲染的会话列表，而不是
  只更新内部状态。
- **历史缓存内存所有权。** 缓存解析不再引用已经释放的文件 buffer，修复了加载缓存行后
  偶发的无效内存问题。
- **线程关闭与过期结果。** AI History worker 现在通过 session mutex 保护共享状态，
  会丢弃过期的转录加载结果，并在销毁时干净地 join。
- **macOS 渲染诊断。** 在 macOS Metal 后端运行时，渲染诊断现在会防护空的 OpenGL 函数
  指针。

## 已知限制

- **AI History 依赖可访问的历史文件。** 如果 Claude Code/Codex 历史目录为空、已移动或
  权限受限，列表会显示较少会话或为空，直到这些来源重新可用。
- **macOS Intel DMG 需要手动触发。** Intel DMG 会在 tag 触发的 release 创建后，由按需
  工作流单独构建并上传。
