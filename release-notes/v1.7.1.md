# WispTerm v1.7.1

> Hotfix release for AI History resume and preview regressions found after
> v1.7.0.

## Fixed

- **macOS AI History resume.** POSIX terminal launches now preserve quoted
  `zsh -lc` and SSH remote-command arguments, fixing `zsh: unmatched '` and
  remote `cd ... && claude --resume ...: No such file or directory` failures.
- **WSL and SSH resume PATH setup.** WSL and SSH AI History resume commands now
  run through the user's shell environment so `codex` and `claude` installed
  from profile-managed paths can be found.
- **Hidden WSL transcript helpers.** WSL AI History scanning and Space preview
  transcript loads now run hidden capture helpers instead of flashing a
  Windows Terminal / console window.

# WispTerm v1.7.1（中文）

> 修复 v1.7.0 发布后发现的 AI History 恢复与预览回归问题。

## 修复

- **macOS AI History 恢复会话。** POSIX 终端启动现在会保留带引号的
  `zsh -lc` 与 SSH 远端命令参数，修复 `zsh: unmatched '` 以及远端
  `cd ... && claude --resume ...: No such file or directory` 失败。
- **WSL / SSH 恢复会话的 PATH 环境。** WSL 与 SSH 的 AI History 恢复命令现在会通过
  用户 shell 环境执行，能够找到安装在 profile 管理路径里的 `codex` 与 `claude`。
- **隐藏 WSL 转录读取 helper。** WSL AI History 扫描和 Space 预览加载转录时，现在使用
  隐藏的后台捕获进程，不再闪出 Windows Terminal / 控制台窗口。
